Linux Commands

  1. File and Directory Management
    1. sudo apt update:connects to the configured package repositories and retrieves the latest package lists.
    2. ls : To List files and directories.
      • ls -l: To List files and directories with timestamp.
      • ls -a : To List files and directories including hidden files
      • touch filename.txt: To create a empty file.
      • vim fileName.txt: create and open a file.If already file created then its opens the file.
      • rm file.txt: To Remove a file.
      • find . -name SecondFile.txt: To find a file the current directory.
      • find / -name "*.log": Search for files and directories
      • du -h --max-depth=1: Disk usage of files and directories.
      • df -h: Show disk space usage of different applications.
      • cp file.txt /new/path/ : copy files
      • cp -r /path/to/source /path/to/destination :Copy directories recursively
      • mv oldFile.txt newFile.txt: Move or rename a file along with its content..
      • mv /path/to/file /new/path : move directory.
      • Mount:When and why to use it
        • The mount command is utilized when you need to access files or directories from a device (like a hard drive, CD-ROM, or network location) on your system.
        • It's especially useful for accessing partitions on a hard disk, external storage devices, or file systems on remote machines.
        • To use mount, you typically need to provide the device identifier and the mount point. mount /dev/sda1 /mnt/mydisk
        • unmount /mnt
    3. mkdir dirname : To create a Directory
      • mkdir -p /path/to/new_dir : Create parent directories if they don’t exist i.e create a directory in particular location
      • cd /path/to/directory : To go that directory.
      • cd ~ :Go to home directory
      • pwd : Print working directory (current location).
      • rm -rf /path/to/directory: Remove directory and its contents recursively.
  2. File Permissions and Ownership
    1. stat fileName.txt : The ‘stat’ command is used to display detailed information about files and file systems, including attributes like file size, permissions, ownership, timestamps, and more in the current directory.
    2. ls -l: To view the permissions of each file and directories in the current directory.
    3. chmod: Change file permissions.
      • chmod 754 fileName.txt: Owner can read/write/execute, others can read/execute, group can read.
      • chmod +x file.sh : Make file executable.
      • chmod 444: Read/Write/Execute owner others and group.
    4. chown: Change file Ownership
      • chown user:group file.txt: Change ownership of file.
      • chown -R user:group /path: Change ownership recursively.
    5. chgrp : Change group Ownership
        The `chgrp` command in Linux is used to change the group ownership of a file or directory. All files in Linux belong to an owner and a group. You can set the owner by using “chown” command, and the group by the “chgrp” command.
      • chgrp [OPTION]… GROUP FILE…: First we need to have administrator permission to add or delete groups. We can login as root for this purpose or use sudo.
      • sudo addgroup groupName : To add a new group
      • sudo chgrp groupName abc.txt: To change the group ownership of a file.
      • chgrp developers file1.txt file2.txt file3.txt: The ‘chgrp’ command can also handle multiple files at once.
  3. TextFile Operations
    1. To Create a file :
      • touch filename.txt
      • vi filename.txt
      • > filename.txt
    2. cat filename : Display file content
    3. clear: To clear the teriminal
    4. tail fileName.txt: Display the last lines of a file
    5. tail -F BigFile.txt
    6. tail -n 4 BigFile.txt
    7. tail -4 BigFile.txt
    8. tail BigFile.txt naga.txt
    9. head fileName.txt: Display the first lines of a file
    10. less fileName.txt: View large files one page at a time
    11. grep: Search text using patterns
      • grep "man" BigFile.txt
      • grep "is " awk.txt
    12. sed : Stream editor, used for searching, replacing text while displaying but not in the file content
      • sed 's/old/new/g' file.txt
      • sed 's/gagan/gagana Sindhu/g' naga.txt
    13. awk : Text processing and reporting
      • awk '{print $1}' naga.txt
      • awk '{print $1, $2}' awk.txt
      • awk '{print $NF}' awk.txt
    14. history > history1.txt : To print history content in history1 text file.If history1.txt file does not exist it will create the page and print the content.
    15. history >> history1.txt :history content will append in history1.txt file.
  4. Process Management
    1. ps: To display list of processes
    2. ps -ef | grep process_name : # Find a specific process.
    3. top :# View system processes and resource usage
    4. kill : Terminate processes by PID.
      • kill PID : # Send termination signal to a process
      • kill -9 PID : # Force kill a process
      • pkill -f process_name :Kill processes by name.
    5. systemctl : Start, stop, enable, or check status of services
    6. systemctl start nginx
    7. systemctl stop service_name
    8. systemctl restart service_name
    9. systemctl status service_name
    10. service : Manage system services
    11. service nginx restart
  5. Shell Script and automation
    1. #!/bin/bash: Shebang line to specify the script interpreter
    2. Variables: VAR_NAME="value"
    3. Conditionals: if [ condition ]; then ... fi
    4. oops: for i in {1..5}; do ... done
    5. function function my_function { echo "Hello" }
    6. bash script.sh: Start a new shell or run a script.
    7. Crontab : Schedule jobs to run at specific times
      • Crontab -e: Edit cron jobs
      • 0 0 * * * /path/to/script.sh : Run script every day at midnight
  6. User and Group Management
    1. sudo : Run commands as another user (usually root) sudo apt update
    2. users: To see current users in linux machine.
    3. sudo useradd userName :Add a new user.
    4. sudo passwd username : Set or change a user's password.
    5. cat /etc/passwd: To see the list of users in the current machine
    6. sudo userdel username: Remove the user
    7. sudo userdel -r username: Remove the user and their home directory
    8. groupadd groupName: Create a new group.
    9. usermod -aG groupName username : Add user to a group.
  7. Networking commands
    1. adding ip addresses in hosts file
    2. ifconfig / ip: Configure network interfaces.
      • ip address show: To show the Ip Address.
      • ifconfig : Deprecated, but still used in older systems.
    3. ping : Check network connectivity . ping google.com
    4. netstat / ss: Network statistics and socket information . ss -tuln
    5. curl / wget: Download files or interact with APIs . curl http://example.com`
      • curl http://example.com # Fetch content of a webpage
      • curl -I http://example.com # Fetch HTTP headers
    6. nslookup : Query DNS information . nslookup google.com
    7. traceroute : Show the path packets take to a network host traceroute google.com
    8. scp : Securely copy files between servers
      • scp file.txt user@remote:/path/to/destination
      • scp -r /local_dir user@remote_host:/remote_dir : copy directories
    9. ssh : Securely connect to a remote system. ssh user@server.com
    10. rsync -avz /local_path user@remote_host:/remote_path . Sync files between local and remote systems.
    11. nmap : Network scanning and security auditing. nmap -sP 192.168.1.0/24
    12. wget: Download files from the web. wget http://example.com/file.zip
  8. Package Management
      Debian/Ubuntu(APT)
    1. apt update: Update the list of available packages. sudo apt update
    2. apt upgrade: Install the latest versions of all packages.. sudo apt upgrade
    3. apt install: Install a package sudo apt install package_name
    4. apt remove: Remove a package. sudo apt remove package_name

    3. tar -czvf archive.tar.gz /path : # Create a compressed archive
  10. gzip file.txt : Compress files.
  11. Search And Filters
    1. grep 'search-term' file.txt ps aux | grep nginx # Find processes related to nginx : Search for a pattern in files or output.
    2. find /path -name "*.log" :Search for files in a directory hierarchy.
    3. locate filename: Find files quickly using a pre-built index.

    1. uptime: Show how long the system has been running. uptime
    2. free: Display memory usage . free -h
    3. vmstat: Report virtual memory statistics vmstat 1 : # Display memory, CPU, and I/O stats every second
